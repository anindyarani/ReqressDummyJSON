Overview
This project focuses on automation testing of the ReqRes API using Java (minimum Java 11). Testing is performed on the Get User endpoint, with the following scenarios:

Get User with Valid Data – Verifies the API returns correct user data.
Get User with Invalid Data – Ensures the API handles invalid requests gracefully.
Get User with Parameters (Pagination) – Tests retrieving users with pagination parameters.

Automation ensures consistent validation of API responses, status codes, and response structure.

Tools & Technologies
Language: Java 11+
Framework: REST Assured / Java REST framework
Build Tool: Maven
Testing: Automation focused


Notes
Project is focused entirely on automation testing.
Java 11 or higher is required to run the project.
Testing covers key Get User endpoint scenarios: valid data, invalid data, and pagination parameters.
